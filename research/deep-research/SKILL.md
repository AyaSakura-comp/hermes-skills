# Deep Research 遞迴研究法 (V2.1 - Todo Optimized)

## 觸發條件
- 當使用者要求 "deep research", 「深度調查」或「盡職調查 (Due Diligence)」時。
- 當單次搜尋無法完整回答，且結果中出現多個需要進一步追蹤的關鍵點時。

## 核心邏輯 (Recursive Loop)
本技能要求 Agent 使用內建 **`todo` 工具** 進行動態任務管理，並遵循以下核心原則：
- **二次研究迴圈 (Secondary Research Loop)：** 當第一份研究報告（初稿或初步彙總）產出後，必須將該報告視為新的「研究輸入」。針對報告中新提取的關鍵資訊、新發現的實體或未釐清的細節，應主動啟動第二輪（甚至更高階）的 `deep-research` 流程，實現「報告 $\to$ 重新定義 $\to$ 二次下鑽」的深度遞歸。
- **遞迴流程：** `Refine -> Todo(Plan) -> Search -> Log(Findings) -> Todo(Update Gaps & New Branches) -> Recursive Dive -> Synthesis`

## 執行步驟

### 1. 初始化任務與知識庫 (Initialization)
- **內建 Todo 管理：** 立即調用 `todo` 工具，將研究目標拆解為初步的任務清單（如：背景掃描、主要技術點調研、競爭對手分析等）。
- **建立 Artifact：** 建立 `research_log.md` 作為**長期知識庫**，記錄已確定的事實、數據與來源 URL，而非僅僅是進度。

### 2. 術語優化與初始掃描 (Query Refinement)
- 使用 `web_search` 獲取主題概況。
- **關鍵步驟：** 從初步結果中提取該領域的「專業術語 (Jargon)」或「產業代碼」。
- **優化指令：** 更新 `todo` 任務，利用提取的術語重新構建搜尋指令。

### 3. 缺口識別與三角論證 (Gap Identification & Triangulation)
- 分析內容，並在 `research_log.md` 中標註：
    - **已知 (Known):** 已有明確結論。
    - **衝突 (Contradiction):** 不同來源說法不一。
- **三角論證要求：** 對於核心事實，必須尋找 **至少 3 個獨立來源**。
- **產地驗證模式 (Origin Verification)：** 當任務涉及「非某國製造」時，必須區分「品牌國籍 (Brand Origin)」與「實際產地 (Country of Origin)」。應優先搜索供應鏈報告、工廠分佈圖或產品底部的標記資訊，而非僅依賴品牌宣傳。
- **動態更新 Todo：** 若發現新缺口、來源衝突或任何值得進一步研究的資訊點，立即調用 `todo` 工具新增「驗證 XXX」或「下鑽 YYY」的新任務，確保研究路徑隨著發現動態演進。

### 4. 遞迴下鑽與邊際價值評估 (Recursive Dive)
- **自主執行路徑：** Agent 應將 `todo` 清單視為行動指南，在每一步完成後，**無需等待使用者指令**，應立即根據清單中的下一個 `pending` 任務自動啟動下一輪研究流程 (Search -> Log -> Update Todo)。
- **並行加速 (Sub-agenting)：** 對於複雜或獨立的 `todo` 項目，Agent 應主動調用 `delegate_task` 開啟子代理 (Sub-agent) 並行執行。子代理需將研究結果、數據來源與分析透過 ACP 回傳至主代理，由主代理統一整合至 `research_log.md`。
- **動態調整：** 按照 `todo` 清單順序執行，並根據新發現動態增刪任務。
- **邊際價值評估：** 隨時檢查 `todo` 清單中的子任務，若繼續挖掘該分支對最終目標貢獻極低，應將該任務標註為已完成或取消（Pruning）。

### 5. 結構化綜合合成 (Structured Synthesis)
- 當 `todo` 清單內的核心任務皆已勾選完成，或達到限制時，產出最終報告：
    1. **執行摘要 (Executive Summary)**
    2. **詳細分析 (Structured by Topics)**：標註來源代號 [1][2]。
    3. **三角驗證結果**：區分「高度確信」與「單一來源」資訊。
    4. **矛盾與未解之謎**：明確標註缺失部分。
    5. **參考文獻 (References)**：**必須詳列所有研究過程中引用的網址 (URLs) 與來源資訊，確保可追溯性。**

## 陷阱與防範 (Pitfalls)
- **同溫層效應：** 避免只採用同一立場的來源。若主題具爭議性，必須搜索反對觀點。
- **資訊過載：** 僅記錄對達成目標有貢獻的資訊，不要做無意義的資料堆疊。
- **忽視內建工具：** 務必保持 `todo` 清單與實際執行進度同步。

## 驗證步驟
- [ ] 是否已調用內建 `todo` 工具管理搜尋路徑？
- [ ] `research_log.md` 是否完整記錄了事實與來源，而非只有任務名？
- [ ] 是否針對關鍵事實進行了至少 3 個來源的交叉驗證？
- [ ] 最終報告是否包含「矛盾與不確定性」的說明？
